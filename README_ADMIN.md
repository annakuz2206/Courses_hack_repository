# 🎯 ITAM Hackathon Platform - Админ-панель

> Полноценная система управления хакатонами с аналитикой, God Mode и экспортом данных

## 🚀 Что это?

Платформа для поиска команд на хакатоны с мощной админ-панелью для организаторов. Участники свайпают карточки команд/людей (как Tinder), а организаторы управляют всем процессом через удобный интерфейс.

## ✨ Основные возможности

### Для участников
- 📱 Авторизация через Telegram
- 📝 Заполнение профиля (роль, навыки, опыт)
- 🎯 Выбор хакатона
- 💫 Свайп команд или участников
- 💬 Чат после матча
- 👥 Создание своей команды

### Для организаторов
- 📊 **Аналитика в реальном времени**
  - Общая статистика участников и команд
  - Распределение по ролям
  - Размеры команд
  - Автоматические рекомендации
  
- 🎪 **Управление хакатонами**
  - Создание и редактирование
  - Настройка дат и размера команд
  - Архивирование
  
- 🎮 **God Mode**
  - Ручное добавление участников в команды
  - Помощь одиночкам найти команду
  
- 📥 **Экспорт данных**
  - Список участников для жюри
  - Список команд с составом
  - Формат готов для CSV/Excel

### Для Super Admin
- 👑 Назначение организаторов
- 🔐 Управление правами доступа
- 🛡️ Полный контроль над системой

## 🏃 Быстрый старт

### 1. Клонируйте репозиторий
```bash
git clone <repo-url>
cd <repo-name>
```

### 2. Запустите Backend
```bash
cd itam_hack
pip install -r requirements.txt
python main.py
```
Backend: `http://localhost:8000`

### 3. Запустите Frontend
```bash
cd frontendik
npm install
npm run dev
```
Frontend: `http://localhost:5173`

### 4. Войдите как Super Admin
1. Откройте `http://localhost:5173`
2. Введите ID: `super_admin_1`
3. Settings → Админ-панель

## 🔐 Система ролей

```
┌─────────────────┐
│  Super Admin    │  ← Создается при деплое
│  (super_admin_1)│  ← Может назначать организаторов
└────────┬────────┘
         │ назначает
         ↓
┌─────────────────┐
│   Organizer     │  ← Управляет хакатонами
│   (u1, u2, ...) │  ← Видит аналитику
└────────┬────────┘  ← Использует God Mode
         │ управляет
         ↓
┌─────────────────┐
│      User       │  ← Обычные участники
│   (u3, u4, ...) │  ← Свайпают и ищут команды
└─────────────────┘
```


### Организатор
```
Вход → Settings → Админ-панель → [Управление] → Аналитика → Действия
```

## 🔧 API Endpoints

### Публичные
- `POST /api/auth/telegram` - Авторизация
- `GET /api/participants` - Список участников
- `GET /api/teams` - Список команд
- `POST /api/swipe` - Свайп

### Админские (organizer/super_admin)
- `GET /api/admin/hackathons` - Хакатоны
- `POST /api/admin/hackathons` - Создать хакатон
- `GET /api/admin/analytics` - Аналитика
- `POST /api/admin/add-to-team` - God Mode
- `GET /api/admin/export/*` - Экспорт

### Super Admin только
- `POST /api/admin/set-organizer` - Назначить организатора

**Полная документация:** `http://localhost:8000/docs`

## 💡 Ключевые фичи

### 1. Автоматические рекомендации
Система анализирует данные и подсказывает:
- "Много участников без команды" → Организовать networking
- "Дисбаланс ролей" → Искать дизайнеров
- "Много неполных команд" → Использовать God Mode

### 2. God Mode
Организатор может вручную добавить участника в команду:
```javascript
await adminAPI.addToTeam('u1', 't1');
// Аня добавлена в команду AI Ninjas
```

### 3. Экспорт для жюри
Одна кнопка → CSV со всеми командами и участниками

### 4. Реальная аналитика
- Распределение по ролям (графики)
- Размеры команд
- % укомплектованности
- Динамика в реальном времени
